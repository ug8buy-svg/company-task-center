我要製作「老闆待辦清單」功能頁面。
這個頁面需要 PIN 碼才能進入（PinGate 已經做好，直接套用）。

【功能說明】
這個清單是給老闆確認用的。
我（管理者）負責新增和刪除項目，老闆負責打勾確認完成。
因為沒有登入系統，所以介面上用兩個區域來區分操作：
- 「管理者區域」：新增、刪除（不需要額外保護，因為整個頁面已經有 PIN）
- 「老闆確認區域」：打勾完成

【資料表】
使用 Supabase 的 boss_todos 表格，欄位如下：
- id：uuid
- content：文字，待辦內容
- is_done：布林值，是否完成
- done_at：時間，打勾時填入
- created_at：時間，建立時間

【頁面功能需求】

1. 顯示待辦清單
   - 未完成的項目排在上方
   - 已完成的項目排在下方，文字變灰色、有刪除線
   - 已完成的項目保留不刪除（留紀錄）
   - 每筆項目顯示：勾選框、內容文字、建立時間

2. 新增待辦項目
   - 頁面上方有一個輸入框 + 新增按鈕
   - 輸入內容後按新增，立即出現在清單裡
   - 輸入框清空，可以繼續新增下一筆
   - 內容不可空白，空白送出要提示錯誤

3. 打勾完成
   - 點勾選框 → is_done 變 true，done_at 填入當下時間
   - 項目移到「已完成」區域，文字變灰、加刪除線
   - 再次點勾選框 → 可以取消完成（is_done 變回 false）

4. 刪除項目
   - 每筆項目右側有刪除按鈕（垃圾桶圖示）
   - 點刪除前顯示確認提示：「確定要刪除這筆待辦嗎？」
   - 確認後從資料庫永久刪除

5. 空狀態
   - 清單是空的時候，顯示「目前沒有待辦事項」的提示

【UI 設計規範】
沿用 prompt_02b 建立的設計風格：
- 使用 src/index.css 裡已定義的 CSS 變數（--bg、--card、--blue 等）
- 支援深色模式自動切換
- 手機版優先，排版清楚易讀
- 字體使用 Noto Sans TC

頁面結構：
- 頂部：返回首頁的按鈕（← 返回）＋ 頁面標題「老闆待辦清單」
- 新增區：輸入框 + 新增按鈕，放在標題下方
- 清單區：
  * 未完成項目（白底卡片）
  * 已完成項目（灰底，文字有刪除線）
- 勾選框樣式：用主題藍色（--blue）
- 刪除按鈕：垃圾桶圖示，hover 變紅色
- 新增按鈕：主題藍色背景，白色文字

互動細節：
- 新增項目時，新項目從上方滑入
- 打勾完成時，項目有淡出再淡入到下方的動畫
- 刪除確認用瀏覽器原生的 confirm()，不需要自製彈窗

【Supabase 操作說明】
- 讀取：從 boss_todos 抓全部資料，按 created_at 排序
- 新增：insert 一筆，content 填使用者輸入的文字
- 打勾：update is_done = true，done_at = 現在時間
- 取消打勾：update is_done = false，done_at = null
- 刪除：delete 這筆資料

【注意事項】
- 修改範圍：src/pages/Todos.jsx
- 不要動其他頁面
- 不要動資料表結構
- 不要動 App.jsx 路由
- 改完請告訴我改了什麼，不要只丟 diff
- 資料從 Supabase 讀取中時，顯示「載入中...」

【完成後請告訴我】
1. 修改了哪些檔案
2. 功能測試結果（新增、打勾、刪除是否正常）
3. 有沒有遇到任何錯誤
