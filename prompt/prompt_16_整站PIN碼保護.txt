公司任務中心 — 開發指令紀錄
指令編號：16
步驟：整站 PIN 碼保護（離職人員阻擋機制）
建立日期：2026/06/XX
================================================================

【背景說明】
原本的展務同仁（仕庭）即將離職，由於本系統沒有帳號登入機制，
任何知道網址的人都能直接打開使用。離職後不希望他能繼續看到
任何頁面內容（包含行事曆、會議記錄、清點表、展務待辦清單）。

需要新增一道「整站 PIN 碼」，在進入首頁之前就先攔截，
跟現有的「劉姐專屬 PIN」（保護待辦清單/備注/看板/帳務）完全分開。

【設計邏輯】

```
打開網址
   ↓
第一關：整站 PIN（新增，所有人都要輸入：老闆、管理者、展務同仁）
   ↓
看到首頁（行事曆、所有功能卡片）
   ↓
第二關：點進劉姐待辦/備注/看板/帳務 → 還要再輸入劉姐專屬 PIN（維持原樣不變）
```

【功能需求】

1. 新增環境變數
   在 .env 新增：
   VITE_SITE_PIN=（我自己填入數字）

   說明：這組 PIN 跟現有的 VITE_PIN_CODE（劉姐專屬用）完全獨立，
   不要共用同一個變數。

2. 建立整站 PIN 保護機制
   - 建立新元件 src/components/SitePinGate.jsx
   - 邏輯與現有的 PinGate.jsx 相同（4位數輸入、錯誤震動、5次鎖定5分鐘）
   - 唯一差異：
     * 解鎖狀態存在不同的 localStorage key（例如 site_pin_unlocked_at），
       跟劉姐 PIN 的 key 要不一樣，避免互相覆蓋
     * 解鎖有效期改為 7 天（604800000 毫秒），不是現有的 24 小時
     * PIN 值從 VITE_SITE_PIN 讀取，不是 VITE_PIN_CODE

3. 套用整站保護
   修改 src/App.jsx：
   - 用 SitePinGate 包住整個路由結構（所有頁面，包含首頁）
   - 確保使用者打開任何網址路徑，都會先卡在整站 PIN 這一關
   - 現有的個別頁面 PinGate（劉姐待辦/備注/看板/帳務/展覽管理）維持不變，
     在整站 PIN 通過之後，這些頁面仍然需要再輸入劉姐專屬 PIN 才能進入

4. PIN 輸入畫面文字
   整站 PIN 輸入畫面標題：「請輸入通行碼」
   （跟劉姐專屬 PIN 畫面的「請輸入 PIN 碼」做文字區隔，避免混淆）

【注意事項】
- 新增檔案：src/components/SitePinGate.jsx
- 修改檔案：src/App.jsx、.env
- 不要動現有的 PinGate.jsx（劉姐專屬用，邏輯不變）
- 不要動資料表結構
- 不要動其他頁面的程式碼
- 改完請告訴我改了什麼，不要只丟 diff
- build 完確認沒有錯誤
