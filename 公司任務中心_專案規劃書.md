# 公司共用任務中心 — 專案規劃書

> 這份文件是給 Claude 看的專案背景說明，方便在新對話裡直接繼續開發。

---

## 一、公司背景

- 公司業務：旅行社、寵物用品、民宿
- 工作模式：遠端為主，只有展覽才見面
- 主要據點：台北（總公司）、台中（我和台中員工）

---

## 二、三個角色與權限

| 角色 | 位置 | 帳號權限 |
|---|---|---|
| **我（管理者）** | 台中 | 新增／編輯／刪除全部內容，唯一有刪除權的人 |
| **老闆** | 台北 | 看全部內容 + 自己的待辦清單可以打勾確認，其餘唯讀 |
| **台中員工** | 台中 | 看展覽清點表 + 參與會議記錄（可即時新增） |

> **台北員工**：目前不需要帳號。台北員工只負責保管和搬運，不需要知道旅展清單內容。若未來有協作需求，再新增帳號即可，不影響現有系統。

---

## 三、功能模組（共六個）

### 1. 老闆待辦清單
- **誰可以看**：我 + 老闆
- **誰可以新增**：只有我可以新增，老闆只能打勾確認
- **內容範例**：
  - 活動贈品需不需要叫貨
  - 住宿券印了沒有
  - 信封袋有沒有準備
  - 其他涉及舞台銷售、老闆需親自確認的事項
- **已完成的項目**：打勾後變灰色保留，不刪除（留紀錄）
- **我可以隨時**：新增一筆、刪除一筆

---

### 2. 老闆專屬備注
- **誰可以看**：我 + 老闆
- **誰可以新增**：只有我
- **內容**：需要老闆決策的事、我想讓老闆注意但不讓員工知道的事
- **台北員工和台中員工**：完全看不到這個區塊

---

### 3. 專案進度看板
- **誰可以看**：我 + 老闆
- **誰可以新增／修改**：只有我
- **三個專案各一張卡片**，每張卡片包含：
  - 目前狀態標籤（進行中 / 等待中 / 卡關 / 尚未開始）
  - 最新進度一句話說明
  - 里程碑清單（可隨時新增、刪除、打勾）
  - 下一個死線日期
- **已完成的里程碑**：打勾變灰保留，不消失
- **注意**：每次展覽準備的東西可能不同，里程碑要可以彈性增減

**三個專案里程碑參考：**

| 專案 | 里程碑範例 |
|---|---|
| 民宿官網 | 需求確認 → 現場拍照 → 影片剪輯 → 文案撰寫 → 網站上線 |
| 旅遊展舞台 | 腳本定稿 → 用品清點 → 主持排練 → 展覽執行 → 展後檢討 |
| 寵物展 | 官網規劃 → 商品上架 → 行銷方案 → 展場設計 → 展覽執行 |

---

### 4. 展覽清點表
- **只有一份清單**，給台中員工使用
- **使用情境**：台中員工出發去台北取貨當天，邊搬邊對清單，確認每樣東西都有拿到、搬上車
- **台北員工**：完全不看這份清單（他們只負責保管和搬運，不需要知道清單內容）
- **誰可以看進度**：我 + 老闆可以看到目前勾選進度
- **每次展覽建新的一份**，品項可以跟上次不同
- **舊的清單**：收進「歷史清單」，可以回頭查，不消失
- **注意**：每次展覽要準備的東西可能稍有不同，清單內容要可以彈性調整

---

### 5. 會議記錄（三方即時共用）
- **參與者**：老闆、我、台中員工（三方線上開會）
- **誰可以新增**：三方都可以即時新增條目
- **每場會議**是獨立一頁，包含：
  - 會議名稱、日期
  - 條目列表（每筆顯示是誰新增的）
- **顯示格式範例**：
  ```
  旅展動線改成L型，入口移到右側    台中員工
  下次贈品改成面紙，數量500個      老闆
  舞台背板下次換直式              我
  ```
- **會議結束後**：記錄保留，可以回頭查

---

### 6. LINE 通知（自動推送）
- **觸發時機**（範例）：
  - 展前 X 天倒數提醒
  - 清點表有未完成項目提醒
  - 老闆待辦清單有未確認項目提醒
- **推送方式**：LINE Notify（免費），發到各自 LINE 或群組
- **由我設定觸發條件**

---

## 四、「已完成」與「歷史」的統一邏輯

| 狀況 | 處理方式 |
|---|---|
| 老闆待辦打勾完成 | 變灰留著，保留紀錄 |
| 里程碑打勾完成 | 變灰留著，保留紀錄 |
| 清點項目打勾完成 | 進度條更新，項目保留 |
| 展覽結束，清單封存 | 移到「歷史清單」，可查閱 |
| 真的要刪除資料 | 只有我（管理者）有刪除權限 |

## 五、技術決定

| 項目 | 選擇 | 原因 |
|---|---|---|
| **資料庫** | Supabase | 免費版夠用，已連接 Claude.ai |
| **前端** | React（網頁） | 可在手機、電腦開啟 |
| **部署** | Vercel | 免費，產生公開網址 |
| **通知** | LINE Notify | 免費，整合進現有 LINE 使用習慣 |
| **登入** | Supabase Auth | 各角色各自帳號密碼 |

---

## 六、建置順序（規劃中）

1. **Supabase 資料庫結構建置**（決定要存哪些資料表）
2. **登入系統**（三個帳號、各自權限）
3. **老闆待辦清單**（最單純，先做這個）
4. **專案進度看板**
5. **展覽清點表**
6. **會議記錄**
7. **LINE 通知**
